Columns

Columns

Each column has horizontal padding (called a gutter) for controlling the space between them.

In a grid layout, content must be placed within columns and only columns may be immediate children of rows.

Column widths are set in percentages, so theyâ€™re always fluid and sized relative to their parent element.

Add the column inside the row. Add a number of columns that add up to 12. If you don't specify a number value, Bootstrap divides by 12 to align the columns.

It doesn't need to be a <div>, any tag will work.

Example of auto-assigned widths

Since this example does not use numbers, it splits the 3 columns equally.

<section class="container">
      <div class="row">
      <article class="col">
        code for column 1
      </article>

      <article class="col">
        code for column 2
      </article>

      <article class="col">
        code for column 3
      </article>
      </div>
    </section>

How to specify the size of the column

When you create a column, you can assign it two values:

  • The breakpoint at which it should become responsive
  • How many of 12 column spaces should it occupy

Since this breakpoint is 'sm', the column will take up the whole width of the browser until it reaches the 'sm' breakpoint (which is 576px or higher).

<div class="col-sm-4">

Offset columns

This is a way to center columns in a larger space. For example, create a 3 column row.

The problem with this code is that the 3rd column is missing. So if you view the site, a blank space is shown where the 3rd column should be.

In addition to column clearing at responsive breakpoints, you may need to reset offsets.

Use offset-0 for this.

<div class="row">
  <div col class="col-4 offset-2">
    code for column 1
  </div>
  <div col class="col-4 offset-sm-2 offset-sm-0">
    code for column 2
  </div>
</row>

If you use offset-sm-0, you must include the breakpoint or it won't work.

This is an older way of centering elements. In Bootstrap 4, you should use alignment instead.

Nesting columns

To nest columns, you simply create a new row inside an existing column. Now, this will create a new regular 12-column grid inside that existing column.

It may be a good idea to use the no-gutters class (as mentioned above) with nested columns. This removes the default 15px padding.

<div class="row">
  <div class="col-sm-9">
    Level 1: .col-sm-9
    <div class="row">
      <div class="col-8 col-sm-6">
        Level 2: .col-8 .col-sm-6
      </div>
      <div class="col-4 col-sm-6">
        Level 2: .col-4 .col-sm-6
      </div>
    </div>
  </div>
</div>

Changing the column order

Even though the columns may be coded in a specific order, you can use the order class to re-arranged them.

<section class="container">
  <div class="row">
    <article class="col-sm-6 order-2">
    code for column 1
    </article>

    <article class="col-sm-6 order-1">
    code for column 2
    </article>
  </div>
</section>

Example

code for column 1
code for column 2

If you do this, make sure to assign a value to all columns, otherwise the order could get messed up.

You can also use .order-first (instead of order-1) and .order-last (instead of order-12).

Setting a single column width

You can set the width of one column and have the sibling columns automatically resize around it. Other columns will resize no matter the width of the center column.

<div class="container">
      <div class="row">
        <div class="col">
          1 of 3
        </div>

        <div class="col-6">
          2 of 3 (wider)
        </div>

        <div class="col">
          3 of 3
        </div>
      </div>
    </div>

Example

1 of 3 (fills in remainder of 3 columns)
2 of 3 (assign 6 columns)
3 of 3 (fills in remainder of 3 columns)

Equal-width multi-row (force column break)

You can force columns to break to a new line by using w-100

Create equal-width columns that span multiple rows by inserting a .w-100 where you want the columns to break to a new line.

This creates a single row with 4 columns, but since w-100 is used, it changes the layout to 2 columns stacked on another 2 columns.

<div class="row">
      <div class="col">col1</div>
      <div class="col">col2</div>

      <div class="w-100"></div>

      <div class="col">col3</div>
      <div class="col">col4</div>
    </div>

Forcing column break at specific breakpoints

You can force columns to break at a specific breakpoint using d-none d-md-block

This hides the breakpoint until the screen enlarges to the md size. It then breaks the columsn into the second row.

Column wrapping

If more than 12 columns are placed within a single row, each group of extra columns will, as one unit, wrap onto a new line.

<div class="row">
  <div class="col-9">.col-9</div>
  <div class="col-4">.col-4<br>Since 9 + 4 = 13 > 12, this 4-column-wide div gets wrapped onto a new line as one contiguous unit.</div>
  <div class="col-6">.col-6<br>Subsequent columns continue along the new line.</div>
</div>

Margin utilities

You can use margin utilities like .mr-auto to force sibling columns away from one another.

In this example, the two columns should be side by side, with a 3rd column missing in a blank space.

Adding .ml-auto pushes the 2nd column to the right.

<div class="row">
      <div class="col-md-4 col-md-4"</div>
      <div class="col-md-4 ml-auto col-md-4 ml-auto"</div>
    </div>

Setting multiple breakpoints

You can set a different column configuration for every breakpoint. You do this by adding them all to the same element.

When the page is on a large scree, all 12 numbers are spread across horizontally.

On a slightly smaller screen, the numbers break into 3 columns.

On a smaller screen, the number break into 2 columns.

On a very small screen, the numbers are in a single column.
