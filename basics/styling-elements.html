Badges

One thing about badges is that they are contextual, so they will resize according to the container.

Classes

  • .badge
  • .badge-pill
  • .badge-(color)

Progress bars

Two sets of tags needed

Outer container uses a .progress class

Inner container uses .progress-bar

You can then use .width (25,50,75,100) and .height

You can also add a label by putting in some text between the internal progress bar element.

Progress bar styles

You can use contextual color classes to change color .bg-(color)

Another option is .progress-bar-striped that gives you a bar that has some striped sort of bars on it.

You can animate that with .progress-bar-animated

You can use as many bars as you want by adding multiple progress bar components inside the main progress bar element.

Accessibility Properties

You should use a series of tags to handle accessibility like role="progressbar" to identify this as a progress bar.

You can also use aria-valuenow to store the current value of the bar,

as well as aria-valuemin to hold a minimum value of the element,

and aria-valuemax to hold the maximum value of the element.

List Groups

it's essentially a way of creating sort of iOS, or Android-styled button lists, and it's just a series of classes that gave you that look.

And in order to use it, what you do is create a container with a .list-group class, and that usually goes inside a div or a ul tag.

inside that container, you create a series of elements using .list-group-item

you can also create list-group items with either list-items buttons, or anchor tags.

You can add .active or .disabled

.list-group-item-action - hover effect

Color classes using list-group-item-(color)

Adding badges to a list group

Use .badge classes

Cards

Cards are a new design element in Bootstrap 4 and it can help you lay out your content into card-like containers that look great and have as a small outline around them

Create a container with a class of .card

Create 2nd container with a class of .card-body. This is where content goes

Classes that can be added in the .card-body:

  • card-text
  • card-title
  • card-subtitle
  • card-link
  • card-img

Colors

You can add contextual colors bg-color

Or border-(color)

And .text-(color)

Card content classes

.card-header

.card-footer

Card Images

.card-img

card-img-top - image at top of card that sits outside sort of the border of the main content

card-img-botom - image at bottom

card-img-overlay - in case you want to have some content that sits on top of an image

List Groups

list-group container that goes around the elements

list-group-item for items in the list

list-group-flush - allows you to take the content that is in the list group, and extend it to the edge of the card.

Card layouts

Create a container that is going to have a number of cards in it and that container should get some classes

.card-group - place the cards side by side, and they will have a shared order.

.card-deck - It's pretty much the same thing as card group, puts everything side by side, but it does have a little of a spacing in between the cards.

.card-columns - makes a really interesting layout where cards align similarly to what you would see in Pinterest.

Media Element

It's really common to have layouts where you have images and text side-by-side and you need the images to be in a separate column than the rest of the content. For this task, Bootstrap gives you a couple of media classes and they're much simpler to use now than they used to be in the previous version of Bootstrap.

Create container with a class of .media

.medai-body - for the elements that you want to be off to one side
