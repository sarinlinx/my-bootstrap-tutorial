Classes

  • .badge

Progress bars

Two sets of tags needed

Outer container uses a .progress class

Inner container uses .progress-bar

You can then use .width (25,50,75,100) and .height

You can also add a label by putting in some text between the internal progress bar element.

Progress bar styles

You can use contextual color classes to change color .bg-(color)

Another option is .progress-bar-striped that gives you a bar that has some striped sort of bars on it.

You can animate that with .progress-bar-animated

You can use as many bars as you want by adding multiple progress bar components inside the main progress bar element.

Accessibility Properties

You should use a series of tags to handle accessibility like role="progressbar" to identify this as a progress bar.

You can also use aria-valuenow to store the current value of the bar,

as well as aria-valuemin to hold a minimum value of the element,

and aria-valuemax to hold the maximum value of the element.

List Groups

it's essentially a way of creating sort of iOS, or Android-styled button lists, and it's just a series of classes that gave you that look.

And in order to use it, what you do is create a container with a .list-group class, and that usually goes inside a div or a ul tag.

inside that container, you create a series of elements using .list-group-item

you can also create list-group items with either list-items buttons, or anchor tags.

You can add .active or .disabled

.list-group-item-action - hover effect

Color classes using list-group-item-(color)

Adding badges to a list group

Use .badge classes

Breadcrumbs

you can use these in unordered or ordered lists, as well as in navigation, just with anchor tags within them.
