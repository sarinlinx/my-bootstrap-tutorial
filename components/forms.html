Basic Form classes

When creating a form, put all code withing a <form> tag.

Inside the that, create a container with a class of .form-group. Put a section of form code within this container, then close it. By default it only applies margin-bottom, but it picks up additional styles in .form-inline as needed.

Each section of the form will have its own container with the .form-group class.

The .form-group class can be used with any block element like fieldsets and divs. It adds a little spacing to the bottom of the element. It also sets the group to display block. And it's required for proper layout of some elements.

<form>
  <div class="form-group">
      code ...
  </div>

  <div class="form-group">
      code ...
  </div>

  <button type="submit" class="btn btn-primary">Submit</button>

</form>

Form controls

The .form-controls class is used to style <input>, <selects>, and <textarea> fields.

File inputs (or upload buttons) instead use .form-control-file.

<form>
  <div class="form-group">
    <label for="exampleFormControlInput1">Email input field</label>
    <input type="email" class="form-control" id="exampleFormControlInput1" placeholder="name@example.com">
  </div>

  <div class="form-group">
    <label for="exampleFormControlSelect1">Example select</label>
    <select class="form-control" id="exampleFormControlSelect1">
      <option>1</option>
      <option>2</option>
      <option>3</option>
      <option>4</option>
      <option>5</option>
    </select>
  </div>

  <div class="form-group">
    <label for="exampleFormControlTextarea1">Example textarea</label>
    <textarea class="form-control" id="exampleFormControlTextarea1" rows="3"></textarea>
  </div>
</form>

<div class="form-group">
  <label for="exampleFormControlFile1">Example file input</label>
  <input type="file" class="form-control-file" id="exampleFormControlFile1">
</div>

Checkboxes

Place checkboxes in a container with a class of .form-check.

The checkbox is an <input> element with a class of .form-check-input.

The text next to the checkbox is a label. Create a <lable> container with a class of .form-check-label. Place any text inside this container.

By default, any number of checkboxes that are immediate sibling will be vertically stacked and appropriately spaced with .form-check.

If you want them to display side-by-side, you can use .form-check-inline

<h3>Example</h3>
<div class="form-check">
  <input  class="form-check-input" type="checkbox" value="" id="defaultCheck1">
  <label  class="form-check-label" for="defaultCheck1">
    First checkbox
  </label>
</div>

<div class="form-check">
  <input  class="form-check-input" type="checkbox" value="" id="defaultCheck1">
  <label  class="form-check-label" for="defaultCheck1">
    Second checkbox
  </label>
</div>

<h3>Inline Example</h3>
<div class="form-check-inline">
  <input  class="form-check-input" type="checkbox" value="" id="defaultCheck1">
  <label  class="form-check-label" for="defaultCheck1">
    First checkbox
  </label>
</div>

<div class="form-check-inline">
  <input  class="form-check-input" type="checkbox" value="" id="defaultCheck1">
  <label  class="form-check-label" for="defaultCheck1">
    Second checkbox
  </label>
</div>

Layout

Bootstrap forms will by default stack vertically. Ths is because Bootstrap applies display: block and width: 100% to almost all our form controls

Additional classes can be used to vary this layout on a per-form basis.

Grid Layout

Use a grid layout for form layouts that require multiple columns, varied widths, and additional alignment options.

<form>
  <div class="row">
    <div class="col">
      <input type="text" class="form-control" placeholder="First name">
    </div>
    <div class="col">
      <input type="text" class="form-control" placeholder="Last name">
    </div>
  </div>
</form>

Example

Form row

You may also swap .row for .form-row, a variation of our standard grid row that overrides the default column gutters for tighter and more compact layouts.

<form>
  <div class="form-row">
    <div class="col">
      <input type="text" class="form-control" placeholder="First name">
    </div>

    <div class="col">
      <input type="text" class="form-control" placeholder="Last name">
    </div>
  </div>
</form>

Auto sizing

.col-auto causes your columns to only take up as much space as needed. The column sizes itself based on the contents.

<form>
<div class="form-row align-items-center">
  <div class="col-auto">
    <label class="sr-only" for="inlineFormInput">Name</label>
    <input type="text" class="form-control mb-2" id="inlineFormInput" placeholder="Jane Doe">
  </div>

  <div class="col-auto">
    <label class="sr-only" for="inlineFormInputGroup">Username</label>
    <div class="input-group mb-2">
      <div class="input-group-prepend">
        <div class="input-group-text">@</div>
      </div>
      <input type="text" class="form-control" id="inlineFormInputGroup" placeholder="Username">
    </div>
  </div>

  <div class="col-auto">
    <div class="form-check mb-2">
      <input class="form-check-input" type="checkbox" id="autoSizingCheck">
      <label class="form-check-label" for="autoSizingCheck">
        Remember me
      </label>
    </div>
  </div>

  <div class="col-auto">
    <button type="submit" class="btn btn-primary mb-2">Submit</button>
  </div>
</div>
</form>

Inline Horizontal Forms

Form sections stack by default. To position them horizontally, use the .form-inline class for the form container.

Form controls within inline forms vary slightly from their default states.

  • Controls are display: flex, collapsing any HTML white space and allowing you to provide alignment control with spacing and flexbox utilities.
  • Controls and input groups receive width: auto to override the Bootstrap default width: 100%.
  • Controls only appear inline in viewports that are at least 576px wide to account for narrow viewports on mobile devices.

You may need to manually address the width and alignment of individual form controls with spacing utilities (as shown below). Lastly, be sure to always include a <label> with each form control, even if you need to hide it from non-screenreader visitors with .sr-only.

<form class="form-inline">
  <label class="sr-only" for="inlineFormInputName2">Name</label>
  <input type="text" class="form-control mb-2 mr-sm-2" id="inlineFormInputName2" placeholder="Jane Doe">

  <label class="sr-only" for="inlineFormInputGroupUsername2">Username</label>
  <div class="input-group mb-2 mr-sm-2">
    <div class="input-group-prepend">
      <div class="input-group-text">@</div>
    </div>
    <input type="text" class="form-control" id="inlineFormInputGroupUsername2" placeholder="Username">
  </div>

  <div class="form-check mb-2 mr-sm-2">
    <input class="form-check-input" type="checkbox" id="inlineFormCheck">
    <label class="form-check-label" for="inlineFormCheck">
      Remember me
    </label>
  </div>

  <button type="submit" class="btn btn-primary mb-2">Submit</button>
</form>

Multicolumn forms

You can use .row and .col normally

You can also use form-row - which is a modified version of row, that gives you a little less room in between the columns. The traditional columns for Bootstrap are 32 pixels in between the columns, and sometimes, that's a little bit too much for form elements. So, you can use this form-row, and it gives you a little bit less room in between things.

col-auto - will automatically determine the width of the column depending on the element that you are including

col-form-label - if you are going to use labels, you need to add the col-form-label class to any labels in your layout.

Create input groups

.input-group - you can use them on regular input elements or even in navigation, anywhere where you want a more integrated look to an input field.

you can also add items at the beginning or at the end of the input groups,

  • input-group-prepend - put it at beginning
  • input-group-append - if you want to place the element at the end

.input-group-text - if you want to add any element that you want to place in there, like a button or a checkbox or just some text,

aria-label - for screen readers

sr-only - or you can use a screen reader only class on an element if you just want to hide it from the screen
