Forms

.form-group - It can be used with any block element like fieldsets and divs and adds a bit of spacing to the bottom of the element. It also sets the group to display block. And it's required for proper layout of some elements.

.form-text - or basic form text elements like paragraphs and text that belongs underneath input fields

.form-controls - for basic text input fields, you want to use the form control class. It's meant for inputs, select and text areas.

.form-control-label - you should use on labels belonging to form controls.

form-control-file - optional form control file, specifically for file inputs if you're going to be using those

Checkboxes and Radio classes

.form-check - The main container for input fields related to checkboxes and radio buttons. It's different than using the regular input field groups because it doesn't go inside the input fields themselves, but instead on a container for these form elements.

.form-check-label - is specifically for the labels that belong to these type of fields,

.form-check-input - for the checkbox or radio button input field

.form-check-inline - Checkboxes and radio buttons display as block elements by default. If you want them to display side-by-side, you can use form-check-inline

Size and validation styles

.form-control-sm - a class to make the input field smaller

.form-control-lg - to make it bigger

.form-inline - You also have the ability to make form groups display as an inline element and that would mean that the label and the input field and perhaps any text after that input field will all show up in the same line.

.form-control-(color)

  • form-control--success

Multicolumn forms

You can use .row and .col normally

You can also use form-row - which is a modified version of row, that gives you a little less room in between the columns. The traditional columns for Bootstrap are 32 pixels in between the columns, and sometimes, that's a little bit too much for form elements. So, you can use this form-row, and it gives you a little bit less room in between things.

col-auto - will automatically determine the width of the column depending on the element that you are including

col-form-label - if you are going to use labels, you need to add the col-form-label class to any labels in your layout.

Create input groups

.input-group - you can use them on regular input elements or even in navigation, anywhere where you want a more integrated look to an input field.

you can also add items at the beginning or at the end of the input groups,

  • input-group-prepend - put it at beginning
  • input-group-append - if you want to place the element at the end

.input-group-text - if you want to add any element that you want to place in there, like a button or a checkbox or just some text,

aria-label - for screen readers

sr-only - or you can use a screen reader only class on an element if you just want to hide it from the screen
